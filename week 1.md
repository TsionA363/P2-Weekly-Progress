# Week 2: Introduction to Server-Side Development

## Client-Server Architecture
Client-Server Architecture is the basic structure of web applications where the **client** (browser) sends requests and the **server** responds with data.

- The client can be a browser like Chrome, and the server is a machine that processes requests and sends back responses.
- When you visit a website, your browser sends an HTTP request to the server.
- The server processes the request, runs code (like PHP), and returns HTML, JSON, or other data.

## Server-Side Development
**Server-side development** refers to writing code that runs on the server, not in the browser.

- It handles logic like authentication, database interaction, and processing form data.
- It ensures data is processed securely before being sent to the client.

## Backend Development
**Backend development** is another name for server-side development.

- It focuses on how the system works behind the scenes.
- It manages application logic, databases, and APIs.

## Server-Side Languages
Common server-side languages include:

- PHP  
- Python  
- Java  
- Node.js  
- Ruby  

PHP is widely used because it is simple, fast, and integrates easily with databases like MySQL.

## Web Servers
A **web server** is software that handles HTTP requests and responses.

- Examples include Apache and Nginx.
- The web server executes PHP scripts and sends the output back to the browser.

## Summary
Server-side development is responsible for:

- Processing data  
- Applying business logic  
- Generating dynamic content for users
